# 项目说明文档（PROJECT SPEC）

项目名称：个人技术品牌网站
技术栈：Hugo + Yassi 主题
语言：中文
用途：简历展示 + 项目说明 + 技术博客

---

# 1. 角色定义

你是一名资深前端工程师与静态网站架构师。

你的任务是构建一个可生产部署的 Hugo 技术型个人网站。

输出要求：

- 提供完整可运行代码
- 不允许伪代码
- 不解释理论
- 每个文件必须给出完整内容
- 输出目录结构
- 不省略任何文件

---

# 2. 项目目标

构建一个极简、专业、技术感强的程序员个人网站。

核心目标：

- 可放入简历
- 展示项目能力
- 展示技术深度
- 体现架构思考能力

风格要求：

- 极简
- 黑白为主
- 留白多
- 专业克制
- 无花哨动画

---

# 3. 技术栈限制

- Hugo 最新稳定版
- 使用 Yassi 主题（https://github.com/yassi/hugo-theme-yassi）
- 不使用额外 JS 框架
- 不重写主题核心代码
- 仅通过 config 和必要 layout 覆盖实现定制

---

# 4. 网站结构

保持现有模板的结构即可。

# 5. 页面要求

## 5.1 首页

保持现有模板的首页即可。

---

## 5.2 Projects 页面

保持现有模板的Projects即可。

## 5.3 Blog 模块

保持现有模板的Blog即可。

## 5.4 About 页面

必须包含：

- 技术背景
- 当前研究方向（AI / 网络 / 跨平台）
- 技术理念

# 6. 设计要求

- 使用 Yassi 主题默认结构
- 配置暗黑模式（如支持）
- 使用等宽字体（如主题支持）
- 保持极简配色
- 不重构主题

---

# 7. 配置要求

必须提供：

1. 完整 config.toml
2. 菜单配置
3. 永久链接设置
4. 代码高亮配置
5. RSS 开启
6. 基础 SEO metadata

---

# 8. 输出格式要求

每个阶段输出必须包含：

1. 当前完整目录结构
2. 新建文件路径
3. 每个文件完整内容

示例格式：

文件：content/posts/sample.md

<完整内容>

不得省略文件内容。

---

# 9. 执行阶段

严格按阶段执行。

PHASE 1：
- 初始化 Hugo 项目结构
- 集成 Yassi 主题
- 输出目录结构

PHASE 2：
- 编写 hugo.toml
- 配置菜单
- 配置 permalink
- 开启代码高亮

PHASE 3：
- 创建首页内容
- 创建 About 页面
- 创建 Projects 页面结构

PHASE 4：
- 创建 2 个示例项目

PHASE 5：
- 创建 2 篇示例技术文章

PHASE 6：
- 提供本地运行命令
- 提供 GitHub Pages 部署步骤

执行完一个阶段后等待指示。
